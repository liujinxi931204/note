# MySQL的事务  
当多个用户同时访问同一数据时，一个用户在更改数据的过程中可能有其他的用户同时发起更改请求，为保证数据的一致性状态，MySQL引入了事务  
事务可以将一系列的数据操作捆绑成一个整体进行统一管理，如果某一事务执行成功，则改事务中进行的所有数据更改均会成功，成为数据库中的永久组成部分。如果事务执行时遇到错误，则就必须取消或回滚。取消或回滚后，数据将全部恢复到操作之前的状态，所有数据的更改均被清除
